JavaScript map, forEach, filter & find methods are only applicable on array. They all are used to access the data from an array more efficiently.

although all of them are array methods, there are some major differences among them. such as:

* map, filter or find returns the output in an array, but forEach doesn't return anything.

* If there exists any empty element inside an array, the map, filter or find doesn't execute functions for empty elements. But, forEach will run over those empty elements & provides "undefined".

* map will call the particular function for each elements of the array & change or update all the elements accordingly. On the other hand, filter will only provide the elements that passes the certain condition or function.

* Though map & filter can provide multiple elements based on original array or conditions imposed, on the contrary the find method will always provide a single (the first matched) element as the output.

Usage of localStorage or sessionStorage & even the cookies are so effective in regards of 'Web Storage API'. Webpages & servers can use them collectively to make the user experience much fluent & interactive as all of them work on users personal preferences such as settings or log-in credentials autosaving, etc.

There are some major differences among them in terms of their mode of action.

* Both in localStorage or sessionStorage data can only be saved onto the browser or user's local memory while, cookies can use both the browser & the server storage.

* Cookies or localStorage data remain saved until the user clears it manually, while sessionStorage data are cleared immediately after the session ends.

* Data transfer speed are not equal for all of the aforesaid process. Usually data storage capacity for cookies, localStorage & sessionStorage are 4kb, 10mb & 5mb accordingly.

JS event loop is a runtime process model which performs the execution of the code of a javaScript file in the browser or web-page.

There is a call stack used in event loop where the tasks are stacked sequentially. There also might be sub-tasks within a task such as: callbacks. When one stack (with its all sub-tasks) is finished, call-stack automatically calls the next stack. This is known as sequentially or synced execution context.

JavaScript event loop can also handle event queue. It means, javaScript can use 'await' functionality while waiting for the response of a particular operation & meanwhile event loop can execute the other call stacks.

This is how, although being a single-threaded programming language, javascript can handle synchronous & asynchronous operations at a time by using its unique event loop process which incudes the call stack & event queue data structure.
